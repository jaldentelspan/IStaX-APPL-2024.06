QoS ? Map Rule Configuration

This page allows to edit or create a single QoS Map rule at a time. Each rule has a key and an action. The key indicates which fields of the frame will be mapped to the fields specified by and according to the action.

QoS Rule Parameters

Rule Key
Indicates the Key of the rule. Depending on the table, the two aforementioned key types exist. Based on them the following key parameters are available:
PCP Priority Code Point: (0 - 7).
DEI Drop Eligible Indicator: (0 - 1).
DSCP Differentiated Services Code Point: (0 - 63).
CoS ID CoS ID: (0 - 7).
DPL Drop Precedence Level: (0 - 3).
DSCP Differentiated Services Code Point: (0 - 63).
DPL Drop Precedence Level: (0 - 3).
Rule Action
Indicates the Action of the rule. The set of actions is independent of the rule key and the following action parameters are available:
CoS Class of Service: (0 - 7).
DPL Drop Precedence Level: (0 - 3).
PCP Priority Code Point: (0 - 7).
DEI Drop Eligible Indicator: (0 - 1).
DSCP Differentiated Services Code Point: (0 - 63).
CoS ID: CoS ID: (0 - 7).
