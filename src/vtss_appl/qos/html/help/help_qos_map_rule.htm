QoS ? Map Rule Configuration

This page shows a list of QoS Map rules, i.e. possible mappings between keys and actions. Even though there is a rule for every possible key, only the rules whose action is not the default are shown here. The default action is '0' regardless of the action type. In the ? Map, there are two distinct key types:
  • PCP - DEI
  • DSCP
  • CoS ID - DPL
  • DSCP - DPL
Rules that use only the PCP value as a key are simply a sub-category of PCP - DEI rules with DEI = 0, thus they belong in the first table. Rules that use only the CoS ID/DSCP value as a key are simply a sub-category of CoS ID/DSCP - DPL rules with DPL = 0, thus they belong in the first/second table.
Each row describes a user-defined, non-default rule. Each rule has a key and an action. The key indicates which fields of the frame will be mapped to the fields specified by the action. The scope of these rules is the selected map.
NOTE: This is just an overview of the non-default rules. The user can add new ones (make them non-default) or edit existing maps using the Add/Edit buttons. Click on the plus sign (empty map entry) to add a new map rule to the corresponding table.

QoS Rule Parameters

Rule Key
Indicates the Key of the rule. Depending on the table, the two aforementioned key types exist. Based on them the following key parameters are available:
PCP Priority Code Point: (0 - 7).
DEI Drop Eligible Indicator: (0 - 1).
DSCP Differentiated Services Code Point: (0 - 63).
CoS ID CoS ID: (0 - 7).
DPL Drop Precedence Level: (0 - 3).
DSCP Differentiated Services Code Point: (0 - 63).
Rule Action
Indicates the Action of the rule. The set of actions is independent of the rule key and the following action parameters are available:
CoS Class of Service: (0 - 7).
DPL Drop Precedence Level: (0 - 3).
PCP Priority Code Point: (0 - 7).
DEI Drop Eligible Indicator: (0 - 1).
DSCP Differentiated Services Code Point: (0 - 63).
CoS ID: CoS ID: (0 - 7).
