Port Configuration Help

This page displays current port configurations. Ports can also be configured here.

The port settings relate to the currently selected stack unit, as reflected by the page header.

Port
This is the logical port number for this row.
Link
The current link state is displayed graphically. Green indicates the link is up and red that it is down.
Warning
Operational warnings of the port.
: No warnings
: There are warnings, use tooltip to see.
Current Link Speed
Provides the current link speed of the port.
Configured Link Speed
Selects any available link speed for the given switch port. Only speeds supported by the specific port is shown. Possible speeds are:
Disabled - Disables the switch port operation.
Automatic - Port auto negotiating speed and duplex with the link partner and selects the highest speed that is compatible with the link partner.
10Mbps HDX - Forces the port in 10Mbps half duplex mode.
10Mbps FDX - Forces the port in 10Mbps full duplex mode.
100Mbps HDX - Forces the port in 100Mbps half duplex mode.
100Mbps FDX - Forces the port in 100Mbps full duplex mode.
1Gbps FDX - Forces the port in 1Gbps full duplex
2.5Gbps FDX - Forces the port in 2.5Gbps full duplex mode.
5Gbps FDX - Forces the port in 5Gbps full duplex mode.
10Gbps FDX - Forces the port in 10Gbps full duplex mode.
25Gbps FDX - Forces the port in 25Gbps full duplex mode.
Dual-media
If a port is Dual-media, this field selects which of the ports to use. If Auto is selected, both ports can be used, and if both ports has link, the SFP port will be preferred.
Advertise Duplex
When duplex is set as auto i.e auto negotiation, the port will only advertise the specified duplex as either Fdx or Hdx to the link partner. By default port will advertise all the supported duplexes if the Duplex is Auto.
Advertise Speed
When Speed is set as auto i.e auto negotiation, the port will only advertise the specified speeds (10M 100M 1G 2.5G 5G 10G) to the link partner. By default port will advertise all the supported speeds if speed is set as Auto.
Flow Control
When Auto Speed is selected on a port, this section indicates the flow control capability that is advertised to the link partner.
When a fixed-speed setting is selected, that is what is used. The Current Rx column indicates whether pause frames on the port are obeyed, and the Current Tx column indicates whether pause frames on the port are transmitted. The Rx and Tx settings are determined by the result of the last Auto Negotiation.
Check the configured column to use flow control. This setting is related to the setting for Configured Link Speed.
NOTICE: The 100FX standard does not support Auto Negotiation, so when in 100FX mode the flow control capabilities will always be shown as "disabled".
PFC
When PFC (802.1Qbb Priority Flow Control) is enabled on a port then flow control on a priority level is enabled. Through the Priority field, range (one or more) of priorities can be configured, e.g. '0-3,7' which equals '0,1,2,3,7'. PFC is not supported through auto negotiation. PFC and Flowcontrol cannot both be enabled on the same port.
Maximum Frame Size
Enter the maximum frame size allowed for the switch port, including FCS. The range is ?-? bytes.
Excessive Collision Mode
Configure port transmit collision behavior.
Discard: Discard frame after 16 collisions (default).
Restart: Restart backoff algorithm after 16 collisions.
Frame Length Check
Configures if frames with incorrect frame length in the EtherType/Length field shall be dropped. An Ethernet frame contains a field EtherType which can be used to indicate the frame payload size (in bytes) for values of 1535 and below. If the EtherType/Length field is above 1535, it indicates that the field is used as an EtherType (indicating which protocol is encapsulated in the payload of the frame). If "frame length check" is enabled, frames with payload size less than 1536 bytes are dropped, if the EtherType/Length field does not match the actually payload length. If "frame length check" is disabled, frames are not dropped due to frame length mismatch. Note: No drop counters count frames dropped due to frame length mismatch
FEC
FEC is short for Forward Error Correction. It is a technique for controlling errors over an unreliable link. The idea is that the sender adds some extra bits to the frame that allows a receiver to correct bit errors in the received frame. It comes in two flavors:
R-FEC (IEEE802.3 clause 74 - sometimes called Firecode). This is meant for 10G.
RS-FEC (IEEE802.3 clause 108 - stands for Reed-Solomon-FEC). This is meant for 25G and does not work on 10G links. The parameter affects both what is requested during clause 73 aneg and what the port is configured to use if not running clause 73 aneg. If running clause 73 aneg on 10G portsor on 25G ports loaded with 10G SFPs, we always tell the link partner that we support R-FEC. What the end user can control with the fec command is whether we request R-FEC. If either us or the link partner requests R-FEC, the port will end up using R-FEC.
Clause 73 on 25G ports loaded with 25G SFPs is a bit different, because 25G ports must support both R-FEC and RS-FEC. With the fec command, the end user can control whether R-FEC and/or RS-FEC is requested or not (all combinations). If at least one end request RS-FEC, the port will end up using RS-FEC, otherwise if at least one end requests R-FEC, the port will end up using R-FEC, otherwise the port will not use any FEC.
auto: This is the default and means the following:
If a 10G port runs clause 73, R-FEC will be requested.
If a 25G port runs clause 73 with a 10G SFP, R-FEC will be requested.
 If a 25G port runs clause 73 with a 25G SFP, both R-FEC and RS-FEC will be requested.
 If a 25G port is loaded with a 25G optical SFP and clause 73 is not running or the port is forced to 25G, RS-FEC will be enabled.
 If a 25G port is loaded with a 25G DAC cable and speed is forced to 25G, RS-FEC will be enabled.
 If a 25G port is loaded with a 25G DAC cable and speed is forced to 5G or 10G,R-FEC will be enabled.
 Otherwise, no FEC will be enabled.
r-fec: If a 10G or 25G port runs clause 73, only R-FEC will be requested. If a 10G or 25G port does not run clause 73, but is loaded with at least a 10G SFP and the speed is at least 5G, only R-FEC will be enabled. Otherwise, no FEC will be enabled.
rs-fec: This option is only available on 25G ports.
If a 25G SFP is inserted and the port is running clause 73, RS-FEC will be requested.
If a 25G SFP is inserted, and the port is not running clause 73, but will end up in 25G (either through forced 25G or auto ), RS-FEC will be enabled.
Otherwise, no FEC will be enabled.
none: If the port is running clause 73, neither R-FEC nor RS-FEC will be requested R-FEC will not be requested (but remember that this does not mean that the clause 73 aneg will not result in the port running some sort of FEC). Otherwise, the port will not run any FEC.

Consequences of Inserting a Given SFP in a 10G or 25G Port

With the advent of Clause 73 (10G and 25G auto-negotiation - sometimes called "KR") and FEC (Forward Error Correction), it is not trivial to determine what will happen if you insert a particular SFP in a particular port.

In the following, tables will be presented that explain the relationships between a configured speed and FEC mode and the actually achieved speed and FEC mode.

The various SFP types are boiled down into four major SFP families as follows:
  • 10G/25G DAC:
    DAC stands for Directly Attached Cable and covers 10GBASE-CR, 25GBASE-CR and 25GBASE-CR-S type of SFPs.
  • 10G/25G Optical:
    Covers fiber SFPs, that is, 10GBASE-SR, 10GBASE-LR(M), 10GBASE-ER, 25GBASE-SR, 25GBASE-CSR, 25GBASE-LR and 25GBASE-ER type of SFPs.
  • 10G/25G CuBP:
    CuBP stands for Copper Backplane and covers 10GBASE-KR, 25GBASE-KR and 25GBASE-KR-S. This is not an SFP type, but indicates that the Serdes interfaces of the two ends are directly connected.
  • 1G - 5G SFPs:
    SFPs with nominal speeds greater than or equal to 1G but less than 10G. Can be any type of SFP, including optical and CuSFPs.
  • 100M SFPs:
    SFPs with a nominal speed of 100 Mbps.
Each table has eight seven columns, which are:
  • Speed
    The speed as configured through CLI (similar speed names are available in Web).
    Notice: force-clause-73 is not available in Web and not directly available in CLI, but it can be enabled in CLI by first issuing platform debug allow. The reason for this is that under normal circumstances, the switch application makes the right decision on enabling clause 73.
    Only if an SFP is miscategorized for one or the reason or if the user really wants to run clause 73 on e.g. optical SFPs, can it make sense to force-enable clause 73.
  • FEC Mode
    The configured FEC mode.
  • 1G Aneg
    Indicates (with "Yes" or "No") whether 1G aneg (IEEE802.3 clause 28 for CuSFPs, IEEE802.3 clause 37 for other SFPs) will be used.
  • Clause 73 Aneg
    Indicates whether IEEE802.3 Clause 73 aneg will be used.

    "No" means that it will not be used.
    Otherwise, three letters (either 'Y' or 'N') are shown separated by a slash (/):
    • First 'Y' or 'N'
      'Y' indicates that R-FEC is requested in the clause 73 aneg.
      'N' indicates that R-FEC is not requested in the clause 73 aneg.
    • Second 'Y' or 'N'
      'Y' indicates that RS-FEC is requested in the clause 73 aneg.
      'N' indicates that RS-FEC is not requested in the clause 73 aneg.
    • Third 'Y' or 'N'
      'Y' indicates that all speeds (25G and all valid speeds below) are advertised in the clause 73 aneg.
      'N' indicates that all valid speeds at or below 10G are advertised in the clause 73 aneg, that is, 25G is not advertised.
  • Speed
    Indicates (with a speed, "As forced" or "From aneg") what speed the port will end up using.
    "As forced" means that whatever forced speed is configured will also be applied.
    "From aneg" means that whatever comes out of 1G or clause 73 aneg will be configured.
  • R-FEC
    Indicates (with "Yes", "No" or "From aneg") whether the port will be configured to using R-FEC (IEEE802.3 clause 74).
    "From aneg" means that whatever comes out of clause 73 aneg will be configured.
  • RS-FEC
    Indicates (with "Yes", "No" or "From aneg") whether the port will be configured to using RS-FEC (IEEE802.3 clause 108).
    "From aneg" means that whatever comes out of clause 73 auto-negotiation will be configured.
  • Notes
    Indicates (with zero or more numbers) additional comments. The comments are:
    • 1
      Clause 73 is required on 25G DAC/Cu Backplane, so a forced speed violates the standard.
    • 2
      RS-FEC is required by 25G optical SFPs, so this violates the standard.
    • 3
      R-FEC not supported by 25G optical SFPs, so this violates the standard.
    • 4
      Clause 73 not supported by 25G optical SFPs, so this violates the standard.
    • 5
      When a 10G SFP is plugged into a 25G port or a 25G port is forced to 5G/10G mode, the 10G PCS will be used, and that PCS doesn't support RS-FEC.
    • 6
      Clause 73 not supported by 10G optical SFPs, so this violates the standard.
    • 7
      If SFP doesn't support configured speed, an operational warning is displayed in show int * status CLI command and the port will not get link.
    • 8
      If SFP is not a 10G nor 25G SFP, and clause 73 is forced, an operational warning is displayed in show int * status CLI command and the port will not get link.
    • 9
      At most 1G can be auto-negotiated by clause 28/37 aneg.
    • 10
      10/100 forced speeds are not supported on 25G ports and these options won't be configurable through management.
    • 11
      If inserting a CuSFP, a warning will be displayed if using a forced speed. Only 'auto' (clause 28 aneg) is supported.
    • 12
      If forcing a speed lower than the SFP's nominal speed (e.g. forcing 2.5G with a 10G SFP) or inserting an SFP into a port with a lower speed than the SFP's nominal, the port may/will get link, but there's no guarantee that it will be error free, so an operational warning is displayed with show int * status CLI command.
