TSN Configuration Help

When using TAS and PSFP between network elements, it is required to have a common global time reference provided by PTP. When booting the device, it will take some time for a configured PTP application to get locked to the common time reference. It may cause malfunctioning of TAS and PSFP if config-change is issued before PTP time is in a Locked or Locking state. A function which can delay the issue of config-change until PTP is Locked/Locking or a configurable time has passed, can be configured here.

Procedure
Select how to ensure PTP state.
Timeout
Specify the maximal number of seconds to wait before config_change is issued.
PTPport
Specify the PTP port to use for sensing PTP status
